# SQL Exploration Walk-through

Ad-hoc schema discovery and query execution with SignalPilot's exploration tools.

## Schema discovery

Before writing any SQL, understand what's in the database.

### List all tables

<MiniCopyBlock
  label="list tables"
  plain
  code={`Tool: list_tables\nconnection: my-warehouse\n→ analytics.orders (48,291 rows, 12 cols, 3 FKs)\n→ analytics.customers (8,204 rows, 9 cols, 0 FKs)\n→ analytics.products (1,203 rows, 6 cols, 2 FKs)`}
/>

`list_tables` returns a compact one-line-per-table overview: columns, PKs, FKs, row counts. Use it to orient before drilling in.

### Describe a table

<MiniCopyBlock
  label="describe table"
  plain
  code={`Tool: describe_table\ntable: analytics.orders\n→ order_id: bigint, NOT NULL, PK\n→ customer_id: bigint, NOT NULL, FK → customers.customer_id\n→ order_date: date, NOT NULL\n→ total_amount: numeric(10,2), nullable\n→ status: varchar(20), NOT NULL`}
/>

`describe_table` gives full column detail: types, nullability, PK/FK relationships, any PII flags.

### Explore a categorical column

<MiniCopyBlock
  label="explore column"
  plain
  code={`Tool: explore_column\ntable: analytics.orders\ncolumn: status\n→ completed:  31,204 (64.7%)\n→ pending:     9,182 (19.0%)\n→ cancelled:   5,210 (10.8%)\n→ refunded:    2,695 (5.6%)`}
/>

`explore_column` shows distinct values with counts and percentages. Essential for understanding filter conditions before writing a WHERE clause.

### Database-wide overview

<MiniCopyBlock
  label="schema overview"
  plain
  code={`Tool: schema_overview\nconnection: my-warehouse\n→ 12 tables, 312,041 total rows\n→ FK density: 2.3 FKs per table\n→ Hub tables: orders (most connected), customers`}
/>

`schema_overview` gives a database-wide summary — useful for orienting in an unfamiliar schema.

## Join path discovery

If you know two tables but don't know how to join them:

<MiniCopyBlock
  label="find join path"
  plain
  code={`Tool: find_join_path\nfrom: analytics.orders\nto: analytics.products\n→ orders → order_items (via order_id)\n→ order_items → products (via product_id)`}
/>

`find_join_path` uses FK relationships to find join paths up to 6 hops. Eliminates guesswork on unfamiliar schemas.

To understand what join type gives the right row count:

<MiniCopyBlock
  label="compare join types"
  plain
  code={`Tool: compare_join_types\nleft: analytics.orders\nright: analytics.customers\non: customer_id\n→ INNER JOIN: 48,000 rows\n→ LEFT JOIN:  48,291 rows (291 orders have no customer record)\n→ RIGHT JOIN: 48,291 rows (same as LEFT)`}
/>

## Cost-aware querying

Before running an expensive query on BigQuery or Snowflake:

<MiniCopyBlock
  label="estimate query cost"
  plain
  code={`Tool: estimate_query_cost\nsql: SELECT * FROM analytics.large_events WHERE event_date > '2024-01-01'\n→ estimated_cost: $1.24\n→ bytes_processed: 4.8 GB\n→ recommendation: Add a partition filter on event_date to reduce cost`}
/>

If the cost is acceptable, run it:

<MiniCopyBlock
  label="query database"
  plain
  code={`Tool: query_database\nsql: SELECT * FROM analytics.large_events WHERE event_date > '2024-01-01'\n→ [returns first 1000 rows — LIMIT auto-injected]`}
/>

## Multi-column statistics

For data profiling across multiple columns at once:

<MiniCopyBlock
  label="explore columns"
  plain
  code={`Tool: explore_columns\ntable: analytics.orders\ncolumns: [total_amount, quantity, discount_pct]\n→ total_amount: min=0.00, max=12,400.00, avg=182.30, 3 NULLs\n→ quantity: min=1, max=50, avg=2.3, 0 NULLs\n→ discount_pct: min=0.0, max=100.0, avg=8.2, 412 NULLs`}
/>

## Debug a CTE step by step

For complex queries, validate each CTE independently:

<MiniCopyBlock
  label="debug cte query"
  plain
  code={`Tool: debug_cte_query\nsql: |\n  WITH base AS (SELECT ... FROM orders WHERE ...),\n  joined AS (SELECT ... FROM base LEFT JOIN customers ...)\n  SELECT ... FROM joined\n→ CTE 'base': valid, 48,291 rows\n→ CTE 'joined': valid, 48,291 rows (no fan-out)\n→ Final SELECT: valid`}
/>

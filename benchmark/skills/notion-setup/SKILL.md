# Notion Setup Skill

One-time setup: connect Notion MCP, configure search scope and report page.

## 1. Check Existing Config

Read `.claude/notion-config.md` in the working directory.

If it exists with both `search_page_ids` and `verification_page_id`:
- Show current config (page titles + IDs)
- Ask if user wants to reconfigure
- If no ā†’ STOP

## 2. Verify Notion MCP

Call `notion-search` with `query: "test"`.

**If tool exists** (returns results or empty set) ā†’ skip to Section 3.

**If tool not found** ā†’ tell the user to run in a separate terminal:

```
claude mcp add --transport http --scope project notion https://mcp.notion.com/mcp
```

Then restart Claude Code and re-run `/notion-setup`. STOP ā€” remaining steps
need MCP tools.

What this does: registers Notion's hosted MCP server for this project. First
connection opens browser for Notion OAuth. After auth, MCP tools (`notion-search`,
`notion-fetch`, `notion-create-pages`) become available.

## 3. Configure Search Scope

Ask user for Notion page URLs or IDs ā€” these are the root pages whose children
will be searchable for context (meeting notes, data dictionaries, product specs).

For each URL/ID:
1. Extract 32-char hex page ID (strip dashes, ignore URL prefix/suffix)
2. Call `notion-fetch` with `id: "<page_id>"` ā€” verify access, get title
3. On failure: page likely not shared with the OAuth connection. User must
   open page in Notion ā†’ Share ā†’ ensure workspace access. Ask for corrected URL
   or skip.

| User pastes | Extracted ID |
|---|---|
| `https://notion.so/Meeting-Notes-2e1939e2883c80199e24c2d9f208be62` | `2e1939e2883c80199e24c2d9f208be62` |
| `https://notion.so/2e1939e2883c80199e24c2d9f208be62?v=...` | `2e1939e2883c80199e24c2d9f208be62` |
| `2e1939e2-883c-8019-9e24-c2d9f208be62` | `2e1939e2883c80199e24c2d9f208be62` |

## 4. Configure Verification Report Page

Ask user for the parent page where build reports go. Each dbt build that uses
Notion context produces a child page with traceability (context ā†’ SQL decisions).

Extract page ID. Call `notion-fetch` to verify. On failure ā†’ ask for corrected URL.

## 5. Save Config

Write `.claude/notion-config.md` in the working directory:

```
# Notion Context Configuration
# Generated by /notion-setup. Re-run to reconfigure.
#
# search_page_ids ā€” agent searches children of these pages for context.
# verification_page_id ā€” build reports created as children of this page.
# Format: <page_id> | <page_title>

search_page_ids:
  <page_id_1> | <title_1>
  <page_id_2> | <title_2>

verification_page_id: <page_id> | <title>
```

## 6. Confirm

Report: search scope (page titles), verification page (title), and that Notion
context will auto-run at dbt workflow Step 1. Re-run `/notion-setup` to change.
